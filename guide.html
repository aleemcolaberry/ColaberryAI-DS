How to use it

Two paths, depending on whether you write code.

👋 If you're non-technical

You never touch code. You describe what you want in plain English and Claude builds an on-brand, downloadable file (social post, flyer, slide, email, one-pager…).

  1. Open a Claude chat with the system attached — open the Claude Design project, or invoke the colaberry-design skill, or paste the GitHub link.
  2. Start with the magic line: "Using the ColaberryAI design system, …"
  3. Say four things: what (e.g. Instagram post), size (e.g. 1080×1080), the words (headline + CTA), and the vibe.
  4. Refine & download: ask for "3 options," tweak in plain English, then export the file.
Copy-paste starter

"Using the ColaberryAI design system, create a [social post / flyer / slide] at [size] for [topic]. Headline: [your headline]. Add a [call-to-action] button and the logo. Keep it warm and on-brand, no emoji — give me 3 options."

Worked example

"Using the ColaberryAI design system, make a square Instagram post (1080×1080) announcing our AI Architect Accelerator founding cohort. Headline: 'Bring your idea. Leave with a deployed AI system.' Add an 'Apply now' button and the logo. Give me 3 options."

💡 You never need hex codes or fonts — Claude pulls those automatically. Your job is the message and the format.

🛠️ If you're technical

Everything is static and dependency-free — no build step.

Plain HTML

<link rel="stylesheet" href="styles.css">
<script src="_ds_bundle.js"></script>
<!-- components on window.ColaberryDesignSystem_098454 -->

React (Vite / CRA / Next)

import 'colaberry/styles.css';
import { Button } from 'colaberry/components/core/Button';
import { Card }   from 'colaberry/components/core/Card';
  • Reference semantic tokens, never raw hex — var(--brand-accent), var(--surface-page), var(--text-strong), var(--space-6).
  • Dark mode is free — set data-theme="dark" on <html>. The teal primary stays; surfaces/text invert.
  • 20 components ship with TypeScript types (.d.ts) and a usage prompt (.prompt.md) each.
  • Full install steps in IMPLEMENTATION.md; brand rules in BRAND.md.

What's inside

  • 245 design tokens (light + dark) — color, type, spacing, radius, shadow, motion. Exported as tokens.json.
  • 20 components — Button, Badge, Card, Avatar, Input, Textarea, Checkbox, Switch, Accordion, Breadcrumb, Pagination, Tooltip, Popover, Progress, Skeleton, Toast, Table, Dialog, Drawer, Carousel.
  • Foundation guidelines — color ramps, type scale, spacing, radius, shadow, logo & icon rules, data-viz palette.
  • 13 fork-ready templates + a marketing-website UI kit.
  • Full logo system — marks, lockups, mono variants, favicons, app icons, social cards.
  • WCAG 2.2 contrast audit + hand-off & prompting guides.

The golden rule

Start with "Using the ColaberryAI design system, …" and describe the outcome. The system supplies the rest — colors, type, logo, components, voice — so anything you make comes out on-brand the first time.
